It includes:
* PostgreSQL setup SQL
* Node.js + Express backend
* Admin login saved for 1 month
* Receipt upload page
* Receipts admin page
* Receipt Items admin page
* Category page
* Settings page placeholder
* CSV export
* Group By dropdown instead of separate Reports page
* payment_method only, no payment_detail
* follow_up only: reimburse / collect
* Corrections marked simple with corrected = true
To start in GitHub Codespaces or local terminal:
npm install
cp .env.example .env
Then edit .env:
DATABASE_URL=your_railway_postgres_url
ADMIN_PASSWORD=your_admin_password
SESSION_SECRET=any_long_random_text
PORT=3000
Run database setup:
psql "$DATABASE_URL" -f migrations/001_fnebooks_schema.sql
Then start:
npm start
Open:
/login.html
/upload_receipt.html
/receipts.html
/receipt_items.html
/categories.html
/settings.html
For Railway deployment, set variables:
DATABASE_URL
ADMIN_PASSWORD
SESSION_SECRET
NODE_ENV=production
Start command:
npm start
This is the first base version. AI receipt reading is not connected yet; the Settings page is already prepared for adding AI extraction later.
